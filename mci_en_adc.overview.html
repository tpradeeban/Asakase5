Overview

To start a DC test from the ALPG, start the ADC (Analog to Digital Converter) for DC or VS by using the MUT signal of the ALPG.

This function is called the ADC start function.

The use of the ADC start function allows a DC test to be executed in a specific operation state (pattern cycle).

  1. With respect to the hardware configuration of and restrictions for the T5830, ADC is categorized into two types as follows:

    • UT_ADC_TYPE_DC : DC test of pins by DC

    • UT_ADC_TYPE_VS : DC test by VS

    There are three ADCs each for the previously mentioned two ADC types.

    Moreover, for measuring pins, there are 12 DC channels and 12 PPS channels, which are assigned to ADC1 to ADC3 in channel units.

    GroupDC/PPS channel
    ADC11-4
    ADC25-8
    ADC39-12

    1. Hardware restrictions

      DC/PPS channels in the same ADC group cannot be used for measurement at the same time. Based on this restriction, up to three DC/PPS channels can be used for measurement simultaneously.

      Because of the hardware restriction, there are some cases where multiple channels cannot be used for measurement depending on the number of DUTs to be measured in parallel even if there is only one DUT pin or VS to be measured.

      In such a case, there is a need to divide DUTs into some groups and perform measurement for each group.

    2. DUT groups for ADC

      MCI UTL_SendAdc() for setting operation conditions for ADC is used to divide DUTs that can be measured at one time into groups using DUT pin numbers or VS numbers to be measured, based on the information in the socket program.

      For a DC test that uses the ADC start function, measurement must be performed for each of these DUT groups.

      Reference the following flowchart to create a program for performing measurement for each group.

For DC
  |-- Setting the number of samples
  |        UTL_SetDcMeasCnt(Number of samples);     // Sets the number of samples.
  |        UTL_SendDc();                       // Sets DC measurement conditions.
  |
  |-- Setting the sampling count
  |        UTL_SetSamplingRate(Sampling time);
  |        UTL_SendSamplingRate();
  |
  +<pin:>
  |
  |-- Setting the pin condition
  |        UTL_SetPinDcConnect( UT_ON );       // DC connection is Pn = DC here.
  |        UTL_SendPin(Pinlist);          // If the DC channel is different, multiple pins can be measured at one time.
  |
  |-- ADC settings
  |        UTL_SetAdcPgEnable( UT_ON );        // Performs the ADC start from the ALPG.
  |        UTL_SetAdcStartCycle(MUT signal name);  // Specifies the signal name for the ADC start.
  |        UTL_SetAdcHistoryMode( UT_ON );     // Sets the history mode to ON or OFF.
  |        UTL_SetAdcPinList(Pinlist);    // Specifies target pins.
  |        UTL_SendAdc();                      // Creates DUT groups.
  |
  |-- Obtaining the largest DUT group number
  |   gmax = UTL_GetAdcDutGroupMaxNumber( UT_ADC_TYPE_DC );
  |
  |-- for(gnum=0; gnum<=gmax; gnum++)          // Loop on a group basis
  |    |
  |    |-- Resetting the history entry count
  |    |       UTL_ResetAdcHistoryCount( UT_ADC_TYPE_DC );  // Unless the history entry count is reset, the previous data remains.
  |    |
  |    |-- Setting the DUT group to measure
  |    |       UTL_WriteAdcDutGroupNumber( UT_ADC_TYPE_DC, gnum );
  |    |
  |    |-- {If necessary, mask DUTs that should not be measured.}
  |    |       UTL_SetNoRejectionVs();
  |    |       UTL_SetResetExclusionByDutBit();
  |    |
  |    |-- Pattern generation and ADC start during pattern generation
  |    |       UTL_StartFct();
  |    |
  |    |-- Obtaining DUTs that belong to the DUT group number    // DUT cursor
  |    |       dutc = UTL_GetAdcDutGroupDutCursor( UT_ADC_TYPE_DC, gnum );
  |    |
  |    |-- the history entry count
  |    |       count = Sampling count x number of ADC starts
  |    |
  |    |-- while( UT_NOMORE != (dut = UTL_NextDut(dutc)) )  //Loop on a DUT basis
  |         |-- Reading the measured values
  |                 UTL_SetDctReadPinDut ( dut );                     // Specifies the DUT to be read.
  |                 UTL_SetDctReadPinType( UT_PINTYPE_PIN );          // Specifies PIN for TYPE.
  |                 UTL_SetDctReadPinNumber(Pin number);              // Converts the pin number from the pinlist.
  |                 UTL_ReadDctPinAllHistoryData( &data, count );     // Reads as many data items as are set to the history entry count.
  |
  |-- Goes to <pin:> when measuring the next pin

  2. T5503HS2

    1. The ADC start function can be used only for VSs (PPSs).

    2. The function that saves histories cannot be used. Histories are reset every time an ADC start is performed.
