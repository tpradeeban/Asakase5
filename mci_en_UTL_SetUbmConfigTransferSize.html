UTL_SetUbmConfigTransferSize --- Setting the Number of Data Items to Be Transferred to FM

SYNTAX

void UTL_SetUbmConfigTransferSize( UbmConfigHandle h, USlider TransferSize );

ARGUMENT(S)

h : UbmConfigHandle

TransferSize : Number of data items to be transferred to FM. Specify it within the following ranges:

Range of TransferSize
UTL_SetUbmConfigCaptureModeRange
UT_UBM_NORMAL0
UT_UBM_FIFO0 to 147456
UT_UBM_FIFO_2BANK0 to 73728

RETURN VALUE

None

DESCRIPTION

This MCI is used to set the number of data items to be transferred from UBM to FM, to the UbmConfig handle.

Obtain the number of data items to be specified here, as follows: 

  Number of data items = PageSize + StatusBytes

For more information on each item, refer to descriptions of the following FailFilerConfig handles:

  • UTL_SetFailFilterConfigPageSize()

  • UTL_SetFailFilterConfigStatusBytes()

The initial value when the handle is obtained is 0.

ERROR CHECK(S)

  • If an incorrect handle is specified, a runtime error occurs.

REQUIREMENT(S)

  • UbmConfigHandle must be obtained.

RESULT(S)

  • UbmConfigHandle has been obtained. (No change)

  • The number of data items to be transferred to FM has been set to the UbmConfig handle.
