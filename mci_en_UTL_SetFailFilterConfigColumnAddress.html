UTL_SetFailFilterConfigColumnAddress --- Storing the column address value in the FailFilterConfig handle

SYNTAX

void UTL_SetFailFilterConfigColumnAddress(FailFilterConfigHandle h, USlider *addr, USlider cnt);

ARGUMENT(S)

  • h: A FailFilterConfig handle

  • addr: The pointer to an array to store the column address values

  • cnt: The number of array elements of the addr argument

RETURN VALUE

None.

DESCRIPTION

This MCI is used to set in a FailFilterConfig handle an array that can be used to specify the order in which addresses are to be accessed in the UBM. If not specified, the UBM addresses are accessed sequentially, starting at zero.

This MCI should be used only when UT_FF_UBM or UT_FF_UBM_PLUS_FC is specified by UTL_SetFailFilterConfigAction().

When a handle is obtained, no array is set as the initial value.

Refer to Example of using UTL_SetFailFilterConfigColumnAddress().

CAUTION

When this MCI is used, be aware that the counters assigned for ECC and RR regions will not be properly configured and the counts will not be correct. Instead, failures need to be counted manually from the data read back from the Fail DB.

ERROR CHECK(S)

  • A runtime error occurs if the specified FailFilterConfig handle is incorrect.

REQUIREMENT(S)

  • The variable area of the addr argument must hold the size of the cnt argument.

RESULT(S)

  • The column address value is set in the FailFilterConfig handle.
