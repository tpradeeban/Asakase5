(*1) When the interleave mode is (n) WAY, the capacity is 1/n of the preceding amount. 

 Example: When the interleave mode is 2WAY, the capacity is 1/2.

(*2) The conditions apply to the specifiable addresses. (The following n indicates the number of WAYs for the interleave mode:)

  • DDR mode is UT_OFF: Only the addresses of multiples of n can be specified.

  • DDR mode is UT_ON: Only the addresses of multiples of n*2 can be specified.

The meaning of the specified addresses differs according to a tester model.

  • Type-A: (*2) : Indicates a UBM address.

  • Type-B: Other than above: Indicates UBM AP. 

  For this reason, addresses for the Type-A must be specified with a consideration of the interleave mode or double-speed mode.

The conditions under which UBM AP is incremented differs according to a tester model.

  • Type-A: Incremented only in a cycle (C0) which enables UBM.

  • Type-B: Incremented in all subsequent cycles after the cycle (C0) which enables UBM.

    Type-A
    

MPAT  SAMMP1
                  ; UBM AP
    NOP  : C0     ; #0
         : C0     ; #1
         : C0     ; #2
         : C0     ; #3
    NOP  : C0     ; #4
         :        ;    <-- UBM AP is not incremented.
         : C0     ; #5
         :        ;    <-- UBM AP is not incremented.
    NOP  : C0     ; #6
         :        ;    <-- UBM AP is not incremented.
         :        ;    <-- UBM AP is not incremented.
         : C0     ; #7
    STPS
END

    Type-B
    

MPAT  SAMMP1
                  ; UBM AP
     NOP : C0     ; #0
         : C0     ; #1
         : C0     ; #2
         : C0     ; #3
     NOP : C0     ; #4
         :        ; #5   <-- UBM AP is incremented.
         : C0     ; #6
         :        ; #7   <-- UBM AP is incremented.
     NOP : C0     ; #8
         :        ; #9   <-- UBM AP is incremented.
         :        ; #10  <-- UBM AP is incremented.
         : C0     ; #11
     STPS
END
