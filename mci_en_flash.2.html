Hardware Descriptions for Flash Memory Test Function

The flash memory test function consists of the hold register, flash counter, limit register, and fail stack register.

An explanation of individual blocks for the T5771 Flash Memory Test Function

  • Fail stack register

    Stores the accumulated result (PASS or FAIL) of FAIL occurring in the comparison cycle for each pin.

    The initial value for each pin is PASS.

    The value for this register can be read with UTL_ReadFctPin().

  • Hold register

    This register stores the results of each pin in the fail stack register or match detection. The one that is stored depends on the hold mode setting.

    The signal set as the load condition for UTL_SetFlashHoldConditionAllDut() is stored in the hold register for each pin in the specified cycle.

    If all pins of each DUT are set to the PASS or MATCH condition, logical comparison and driver output of the DUTs in the PASS or MATCH condition are disabled as of the next cycle.

    If the hold register is cleared, the disabled logical comparison and driver output are enabled again.

    Hold Register Clear is conducted with cycles output by the pattern generator where signals are set as UTL_SetFlashHoldConditionAllDut() clear conditions.

    The initial value for each pin is FAIL or UNMATCH.

    The value for this register can be read with UTL_ReadFlashHold().

  • Flash counter

    This counter counts the number of write/erase operations for each DUT.

    When Hold Register values are TOTAL FAIL or UNMATCH for each DUT, signals set as UTL_SetFlashCounterConditionAllDut() increment conditions output by the pattern generator are incremented by the Flash Counter.

    If the flash counter matches the limit register, logical comparison and driver output of the matched DUTs are disabled as of the next cycle.

    If the flash counter is cleared, disabled logical comparison and driver output are enabled again.

    The flash counter is cleared in the cycle where the signal set as the clear condition for UTL_SetFlashCounterConditionAllDut() is specified.

    The initial value for the flash counter is 0.

    The value for this register can be read with UTL_ReadFlashCounter().

  • Limit register

    The limit value for the number of write/erase operations for each DUT is set in this register.

    If the value for the flash counter reaches the value for the limit counter, FAIL occurs and the pattern generator stops. This FAIL is referred to as MAXFAIL.

    With UTL_SetFlashLimitAllDut() or UTL_SetFlashLimitByDut(), the value set in this register can be set to the Flash handle.

  • Disabling logical comparison

    The results of logical comparison for all pins are forcibly set to PASS or MATCH.

    Logical comparison and driver output can be disabled for the following DUTs.

    • DUTs in which all pins are matched

    • DUTs where the flash counter matches the limit register

    • DUTs with mask specification

    The disabled condition of logical comparison is retained until the pattern program is completed, unless it is canceled.

    For DUTs where logical comparison has been disabled once, clear the hold register and flash counter to enable logical comparison again.

    If UTL_SetPinPatternInhPerPin() is used, settings for logical comparison of individual I/O pins can be disabled. For details, refer to "FutureSuite MCI Reference Manual 1 - Pin Conditions Interface UTL_SetPinPatternInhPerPin".

  • Disabling driver output

    For each DUT, the output pattern of the pins with UTL_SetPinPatternInh() specification is forcibly set to 0.

    By specifying UTL_SetPinPatternInh() as a pin to be connected to Write Enable (/WE) and Chip Enable (/CE) of the DUT, excessive write or excessive erase for the DUT can be prevented.

    Driver output is disabled for the pins with UTL_SetPinPatternInh() specification of the following DUTs.

    • DUTs in which all pins are matched

    • DUTs where the flash counter matches the limit register

    • DUTs with mask specification

    The disabled condition of driver output is retained until the pattern program is completed, unless it is canceled.

    For the DUTs with which driver output has been disabled once, clear the hold register and flash counter to enable driver output again.

    if UTL_SetPinPatternInhPerPin() is used, settings for driver output of individual I/O pins can be disabled. For details, refer to "FutureSuite MCI Reference Manual 1 - Pin Conditions Interface UTL_SetPinPatternInhPerPin".
